# How to Configure Your Sidebar

## Goal

Customize how your documentation pages are organized and displayed in the sidebar navigation.

---

## Prerequisites

- A Docusaurus site with at least a few Markdown files
- Basic understanding of JavaScript object syntax (for manual config)

---

## How the Sidebar Works

The sidebar is the navigation panel on the left side of your docs that shows all pages organized hierarchically.

## Using Frontmatter for Automatic Sidebar

**With DocuSapiens, this is the easiest approach.**

Use the `sidebar_position` field in each page's frontmatter to control order:

### Step 1: Add sidebar_position to Each File

In `docs/getting-started/installation.md`:

```markdown
---
title: Installation
sidebar_label: Install
sidebar_position: 1
---
```

In `docs/getting-started/first-steps.md`:

```markdown
---
title: First Steps
sidebar_position: 2
---
```

### Step 2: Verify the Order

Docusaurus reads the `sidebar_position` values and orders pages accordingly:

```
Getting Started
â”œâ”€â”€ Install                    (sidebar_position: 1)
â””â”€â”€ First Steps               (sidebar_position: 2)
```

[SCREENSHOT PLACEHOLDER: Sidebar showing sections in correct order based on sidebar_position values]

### Step 3: Adjust Positions as Needed

To reorder, just change the numbers:

```markdown
---
sidebar_position: 5        # Moved to position 5
---
```

Rebuild, and the page moves in the sidebar.

## Advanced: Manual Sidebar Configuration

If you need full control, create/edit `sidebars.js` in your project root:

### Basic Structure

```javascript
module.exports = {
  tutorialSidebar: [
    'intro',  // docs/intro.md
    {
      label: 'Getting Started',
      items: [
        'getting-started/installation',  // docs/getting-started/installation.md
        'getting-started/first-steps',   // docs/getting-started/first-steps.md
      ],
    },
  ],
};
```

### Multiple Sidebars (Advanced)

```javascript
module.exports = {
  // Sidebar for /docs/* pages
  docsSidebar: [
    'intro',
    { label: 'Getting Started', items: [...] },
  ],
  
  // Sidebar for /blog/* pages (if you have a blog)
  blogSidebar: [
    { label: 'Recent Posts', items: [...] },
  ],
};
```

## Sidebar Features

### Collapsible Groups

Groups can be expanded/collapsed:

```javascript
{
  label: 'Getting Started',
  collapsible: true,    // â† Allow collapsing
  collapsed: false,     // â† Start expanded
  items: ['installation', 'first-steps'],
}
```

[SCREENSHOT PLACEHOLDER: Sidebar showing expandable section with arrow icon, showing expanded/collapsed states]

### Custom Icons & Links

Add external links to your sidebar:

```javascript
{
  type: 'link',
  label: 'Visit our site',
  href: 'https://example.com',
}
```

### Autogenerated Category

Auto-detect all files in a folder:

```javascript
{
  type: 'autogenerated',
  dirName: 'api',  // Auto-discover docs/api/ folder
}
```

## Troubleshooting

### âŒ Pages appear in wrong order

**Cause:** Missing `sidebar_position` or wrong numbers.

**Solution:**
```markdown
---
sidebar_position: 1  # Add or fix this
---
```

### âŒ Pages don't appear in sidebar

**Cause:** Page might be ignored or in wrong location.

**Solution:**
- Check page is in `docs/` folder (or subfolder)
- Check page has `.md` extension
- Add `sidebar_position` to frontmatter

### âŒ Sidebar doesn't update after adding page

**Cause:** Build hasn't run yet.

**Solution:**
1. If local: `npm run build` or `npm start`
2. If DocuSapiens: Wait for it to rebuild automatically (or manually trigger)

## Best Practices

### âœ… Do This

- Use `sidebar_position` to order pages
- Use descriptive `sidebar_label` for readability
- Create folders to group related pages
- Keep nesting 2-3 levels max (for usability)

### âŒ Don't Do This

- Rely on alphabetical order (use sidebar_position)
- Have unclear page labels
- Create deeply nested structures (10+ levels)
- Mix auto and manual configuration
